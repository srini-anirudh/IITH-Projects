High Level Specifications

    A.Functional Specifications

    1. The application will have 2 versions - one for Teacher and other for student. There will be ways to login into both type of accounts.
    2. Student version:
      1. Uploading assignments and see automated To-do list of all the courses combined
      2. Online time-bound quiz
      3. View course material uploaded by instructor
      4. Chat option for asking doubts
      5. Viewing grades of courses after it is released by the instructor
    3. Teacher Version:
      1. Evaluating assignment and quizzes. For Quiz there could be an option to prepare a question bank. Randomly generated subset of k out of n (configurable values) questions may be made available to students in a quiz. Similarly, the answers could also be in randomized order (if multiple choice).
      2. Uploading course materials
      3. Posting announcements
      4. Releasing grade after course completion

      B.Technical Specifications

      The project has to be developed as a web application with HTML as front-end, C++ as back-end and the feasibility of using a Database server (mySQL, MSExcel) to be evaluated and used.

      Feasibility of using Open source for Chat Module to be assessed and integrated.
